# 코스닥 무자본 M&A 위험 탐지

공개된 DART 전자공시와 KRX(KIND) 데이터만으로, 코스닥 상장사의 **무자본 M&A**
(자기 자본 없이 상장사를 인수해 CB·유상증자로 자금을 끌어온 뒤 유출시키고
상장폐지에 이르게 하는 행위) 위험 징후를 탐지한다.

KBS 시사기획 창 **'코스닥 개미귀신'** 등 무자본 M&A 기획 보도에서 영감을 받아,
방송이 고발한 수법(경영권 인수 → CB·유상증자 → 자금 유출 → 상장폐지)을
공개 데이터로 정량화·자동화할 수 있는지 검증해 본 프로젝트다.

핵심 관점: **부실 예측이 아니라 "약탈 행위와 배후 인물" 탐지.**
재무비율(회사가 어려운가)이 아니라 공시에 드러나는 행위(누가 인수해서 무엇을
발행했는가)와 그 배후 인물의 이동 경로를 본다.

> ※ 본 프로젝트의 모든 결과는 공개 공시 기반의 **의심 정황 선별**이며,
> 특정 기업·개인의 위법을 단정하지 않는다. 추가 검토용 참고 자료다.

## 1. 접근

### 신호 1 — 위험도 점수 (예측 모델)
- **위험군 정의**: 거래소 상장적격성 실질심사 대상 기업 (2018~, 224개)
- **대조군**: 실질심사 이력이 없는 코스닥 상장사 무작위 224개
- **변수 4개 (전부 "행위" 지표)**: 최대주주·경영권 변경, 유상증자, 전환사채(CB) 발행,
  담보제공계약 — 기준일 이전 3년간 공시 건수
- **모델**: 랜덤포레스트 — **ROC-AUC 0.829 (±0.037)**
- 관찰 창은 2/3/5년을 비교해 3년 채택 (2년 0.808, 3년 0.829, 5년 0.813)
- 변수 중요도(MDI·permutation 교차 확인): 경영권변경·유상증자·CB 세 축이 핵심,
  담보는 미미 — 인수→자금조달로 이어지는 무자본 M&A 수법 순서와 일치
- 학습 표본에는 교차검증(out-of-fold) 점수를 부여해, 대조군을 포함한
  코스닥 보통주 전체가 과적합 없는 위험도를 갖는다 (`data/risk_ranking_all.csv`)

### 신호 2 — 위험 경영진 네트워크
- 실질심사 위험군(2013~, 264개 수집)의 **실질 경영진**(사외이사·감사 제외)을 수집,
  현재 상장사 임원과 이름+생년월로 매칭
- "부실기업을 거친 인물이 지금 어느 상장사에 몇 명 있는가"를 추적
- 재직 시점 창: **사유발생 5년 이내** (0년~무제한 스윕 결과 3~5년이 정점, 7년 이상 하락)

### 검증 — 두 신호는 실제로 예측하는가
정답 = KIND 매매거래정지 목록 중 사고성 정지(상장폐지 사유발생·투자자 보호 등).
무관 사유(스팩 합병, 주식 병합·분할, 조회공시 위반)와 **학습에 쓴 기업은 제외**(순환논리 방지).

| 신호 | lift (무작위 대비) |
|---|---|
| 위험도 상위 (단독) | **약 8.4배** |
| 위험 경영진 재직 (단독, 5년 창) | 약 3.5배 |
| **두 신호 결합** | **약 12배** |

- '투자경고' 사유 포함/제외 민감도 분석: 두 경우 모두 결론 동일 (견고성 확인)
- 위험인물 **인원수** 분석: 다수(2명+)가 몰린 기업은 그룹 계열사 겸직(정상 경영)이
  많아 단독 신호로는 오히려 약함 — 위험도와 결합해야 유효
- 실제 적중 예: 위험도 상위 1% 이내로 예측했던 기업들이 이후 상장폐지 사유
  발생으로 거래정지 (앱 감시 리스트의 '적중 사례'에서 확인 가능)

## 2. Workflow

수집(느림, API)과 분석(빠름, 반복 가능)을 분리했다. 공시는 집계값이 아니라
**공시 한 건당 한 줄(long format)로 5년치**를 저장해두고, 분석 시 원하는 창(N년)으로
집계한다 — 관찰 창 실험에 API 재호출이 필요 없다.

```
[A. 수집 — DART API, 한 번만]
A_1_build_dataset.py            위험군/대조군 목록 (실질심사 2018~ + 무작위 대조)
A_2_build_predict_targets.py    예측대상 목록 (전체 코스닥 - 스팩·우선주 - 학습기업)
A_3_collect_raw_disclosures.py  공시 원자료 5년치 수집 → data/raw_disclosures.csv
A_4_collect_current_execs.py    현재 상장사 임원 수집

[B. 분석 — API 없음, 반복 가능]
B_1_aggregate_features.py       N년 창 집계 (기본 3년, 인자로 조절: python B_1... 5)
B_2_train_model.py              랜덤포레스트 학습 + 변수 중요도
B_3_predict.py                  OOF+전체모델 예측 → data/risk_ranking_all.csv
B_4_validate_signals.py         신호 검증 (lift, 재직 창 스윕, 민감도)

[C. 네트워크]
C_1_build_risky_set.py          네트워크용 위험군 (2013~)
C_2_collect_execs.py            위험기업 임원 수집 (API)
C_3_build_groups.py             인물 공유 기반 기업 그룹화
C_4_match.py                    현재 상장사 임원과 매칭
C_5_analyze_persons.py          핵심 인물 분석 (재직시점별 사전계산)

[앱]
통합앱.py + pages/              조회 대시보드 · 네트워크 분석 · 감시 리스트
```

전체를 순서대로 무인 실행: `python clean_run.py` (약 1시간, 로그 run_all_log.txt)
진단: `python check_coverage.py` — 연도별 임원 수집 성공률 (2013년 이전은 DART 미제공)

## 3. 실행 방법

```bash
pip install pandas scikit-learn streamlit pyvis networkx openpyxl lxml python-dotenv opendartreader
```

1. `.env`에 `DART_API_KEY=발급키` ([opendart.fss.or.kr](https://opendart.fss.or.kr))
2. `input_files/`에 KIND 다운로드 파일 3개:
   `실질심사법인_전체.xls`, `코스닥_상장.xls`, `매매거래정지종목.xls`
3. `python clean_run.py` (또는 A→B→C 순서대로 개별 실행)
4. `streamlit run 통합앱.py`

배포: `python package_install.py` → `실행패키지/` 폴더가 생성되며, 파이썬만 있는
컴퓨터에 통째로 복사해 배치파일로 실행 가능.

## 4. 앱 구성

- **조회 대시보드** — 종목 검색: 위험도(백분위), 매매거래정지 상태(⛔),
  위험 경영진(재직 5년 이내) 표시. 학습 대조군은 교차검증 점수로 표시.
- **네트워크 분석** — 위험기업 간 연결 / 위험기업→현재기업 확산 /
  핵심 인물(여러 부실기업을 거친 경영진) / 현재기업별 위험인물 집계
- **감시 리스트** — 미정지 + 위험도 상위 + 위험인물 재직이 **동시에** 켜진 기업.
  하단에 같은 조건으로 실제 거래정지된 '적중 사례' 표시.

## 5. 설계 판단 기록

- **횡령·배임 공시를 변수에서 제외** — 예측력은 강하지만 위험의 원인이 아니라
  결과(레이블 누수)이므로 제외. "무엇을 위험의 근거로 삼을지는 사람의 판단"
- **재무비율 미사용** — 정상적 사업 실패와 의도적 약탈을 구분하는 것이 목적이므로
  재무 상태가 아닌 행위 지표만 사용
- **관찰 창·재직 창을 실험으로 결정** — 둘 다 "짧으면 놓치고 길면 억측" 패턴이
  독립적으로 재현됨 (지표 3년, 재직 3~5년 정점)
- **검증에서 학습 기업 제외** — 실질심사로 학습한 모델을 실질심사로 채점하지 않도록,
  별개 사건(사고성 거래정지)을 정답으로 사용
- **위험군은 백분위 미산출** — 이미 위험이 확정된 기업이라 예측 점수를 노출하지 않음

## 6. 한계와 향후 과제

- **사명 변경 미추적** — 사명 세탁(예: 옛 사명으로 검색 시 누락)을 못 잡음.
  DART 기업개황으로 이력 매핑 예정
- **법인 최대주주 미포착** — 임원 네트워크는 개인 기준이라, 법인(SPC)을 통한
  지배는 안 잡힘. 최대주주(법인이면 그 실소유주 1-hop) 추적이 다음 과제
- **유행 테마 편승 미구현** — 사업목적 변경 공시의 시기별 키워드 매칭으로 근사 가능
- **CB는 건수만 반영** — 발행 금액·전환가 등 조건은 공시 본문 파싱 필요
- **정답 표본이 작음** — 거래정지 정답 18~19건으로 lift의 절대값보다
  신호 간 상대 비교와 추세를 신뢰할 것
- **거래정지 목록은 스냅샷** — 주기적으로 KIND에서 갱신 필요

## 7. 데이터 출처

- DART 전자공시 OpenAPI (금융감독원) — 공시 목록, 임원 현황
- KIND (한국거래소) — 실질심사 법인, 상장 법인, 매매거래정지 목록
